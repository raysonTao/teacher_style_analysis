## 五、注意事项

### 5.1 材料填写注意事项

1. **一致性**：申请表、使用说明书、设计说明书中的软件名称、版本号必须完全一致
2. **完整性**：所有必需材料必须齐全，缺一不可
3. **真实性**：填写内容必须真实，不得虚假陈述
4. **规范性**：文档格式规范，页码连续，排版整齐

### 5.2 源代码注意事项

1. **连续性**：前30页和后30页必须来自同一软件，不得拼凑
2. **完整性**：代码逻辑完整，能够体现软件核心功能
3. **注释**：包含必要注释，但不要过度注释（影响行数）
4. **敏感信息**：代码中不得包含密码、密钥等敏感信息

### 5.3 常见退回原因

1. 软件名称不规范（使用了公司名、产品类别等）
2. 文档页数不足（使用说明书或设计说明书少于30页）
3. 源代码页数不足或行数不够
4. 源代码前后不连贯，明显拼凑
5. 申请表填写不完整或签字缺失

### 5.4 审查周期

- **普通申请**：30-35个工作日
- **加急申请**：根据选择的加急档次
- **补正**：如材料不符合要求，需补正后重新审查
