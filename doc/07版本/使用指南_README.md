## 🛠️ 工具推荐

### 绘图工具
1. **Draw.io** (https://app.diagrams.net/)
   - 免费、在线、无需安装
   - 适合绘制系统架构图、流程图
   - 导出为PNG或SVG格式

2. **PowerPoint/Keynote**
   - 适合绘制网络结构图
   - 使用"插入 → 形状"和"插入 → SmartArt"

3. **Python (Matplotlib/Seaborn)**
   - 适合绘制实验结果图（柱状图、热图、折线图）
   - 代码可重复，易于调整

4. **TikZ（LaTeX）**
   - 如果你用LaTeX写论文，TikZ可以绘制非常专业的学术图表
   - 学习曲线较陡，但效果最佳

### 公式编辑工具
1. **Word内置公式编辑器**
   - 插入 → 公式
   - 支持基本的LaTeX语法（如 \alpha, \sum, \frac）

2. **MathType**
   - Word的强大插件
   - 支持完整的LaTeX语法
   - 可以直接粘贴LaTeX代码

3. **在线LaTeX编辑器**
   - Overleaf (https://www.overleaf.com/)
   - 可以在线编辑和预览公式
   - 然后截图插入Word

## 💡 写作技巧

### 1. 如何写"本章导读"
**模板：**
```
在第X章中，我们已经...。然而，...仍然是一个挑战。

本章聚焦于...，旨在解决...问题。具体而言，本章的主要内容包括：
1. ...（X.1节）
2. ...（X.2节）
3. ...（X.3节）

通过本章的工作，我们为...奠定了基础。
```

### 2. 如何写"本章小结"
**模板：**
```
本章详细介绍了...，主要完成了以下工作：

1. **问题分析**：...
2. **方法设计**：...
3. **实验验证**：...

实验结果表明，...达到了...的准确率，相比...提升了...个百分点。

**与现有工作的对比**：...

**局限性与未来工作**：...

这些工作为下一章的...奠定了技术基础。
```

### 3. 如何写过渡段落
**模板：**
```
前一章的总结 + 问题引出 + 本章目标

例如：
在第3章，我们已经设计了MMAN的整体架构。然而，要实现有效的多模态融合，首先需要从原始视频中提取高质量的特征表示。**因此，本章聚焦于多模态特征提取的技术细节。**
```

### 4. 如何描述技术模块
**标准结构：**
```
【模块名称】（例如：4.3 跨模态注意力模块）

4.3.1 设计动机
  - 为什么需要这个模块？
  - 现有方法有什么不足？
  - 这个模块如何解决问题？

4.3.2 数学建模
  - 输入是什么？（附维度）
  - 计算过程是什么？（附公式）
  - 输出是什么？（附维度）
  - 设计考量（为什么这样设计？）

4.3.3 网络结构图
  - 【插入图X.X】

4.3.4 实验验证
  - 消融实验：移除该模块后性能下降多少？
  - 与其他设计的对比
```

### 5. 如何写消融实验
**表格模板：**
| 模型配置 | 准确率 | ΔAcc | 说明 |
|---------|--------|------|------|
| 基线 | 85.0% | - | 不包含任何改进 |
| + 模块A | 87.5% | +2.5% | 模块A的贡献 |
| + 模块A + 模块B | 90.0% | +5.0% | 模块B的贡献（+2.5%） |
| 完整模型 | 91.4% | +6.4% | 所有模块的总贡献 |

**分析模板：**
```
从表X可以看出：
1. 模块A单独贡献了X.X%的提升，证明了...
2. 模块B在模块A的基础上进一步提升了X.X%，说明...
3. 完整模型达到了X.X%的准确率，显著优于基线方法
```

## ⚠️ 常见错误

### 1. 公式相关
- ❌ 公式没有编号
- ❌ 公式在正文中没有被引用
- ❌ 变量没有定义（例如：$d_k$ 是什么？）
- ❌ 符号不一致（一会儿用 $W$，一会儿用 $w$）
- ✅ 正确做法：
  - 所有公式编号（3.1, 3.2...）
  - 正文中引用："如公式(3.1)所示..."
  - 首次出现变量时定义："其中，$d_k$ 是注意力维度"

### 2. 图表相关
- ❌ 图表没有标题和说明
- ❌ 图表没有在正文中引用
- ❌ 图表质量低（截图模糊、标注不清）
- ✅ 正确做法：
  - 图表标题："图4.1 MMAN整体架构图"
  - 正文引用："如图4.1所示..."
  - 高分辨率（至少300 DPI）

### 3. 实验相关
- ❌ 只给出总体结果，没有消融实验
- ❌ 只跟单模态对比，没有跟其他融合方法对比
- ❌ 没有误差分析或失败案例分析
- ✅ 正确做法：
  - 消融实验：逐个移除模块
  - 多个baseline对比
  - 参数敏感性分析
  - 失败案例分析

### 4. 写作风格
- ❌ 章节间没有过渡，显得突兀
- ❌ 缺少"本章导读"和"本章小结"
- ❌ 技术描述过于简略，缺少设计考量
- ✅ 正确做法：
  - 每章开头有导读，结尾有小结
  - 章节间有过渡段落
  - 技术描述要解释"为什么"
